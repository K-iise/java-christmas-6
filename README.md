# 미션 - 크리스마스 프로모션

## 🔍 기능 목록 작성

중복된 할인과 증정을 통해서 고객들에게 해택을 제공하는 것을 목표로한다.

0. 주문 가능 여부 검증
- 주문 메뉴가 음료만으로 구성되었는지 검증한다.
- 주문한 전체 메뉴 개수가 20개 이하인지 검증한다.

1. 이벤트 참여 가능 여부 판단
- 할인 전 총 주문 금액이 10,000원 이상인지 판단한다.

2. 날짜 및 이벤트 기간 판단
- 방문 날짜가 1~31 범위인지 검증한다.
- 방문 날짜가 이벤트 적용 기간인지 판단한다.
- 방문 날짜로 요일을 계산한다.

3. 12월 이벤트 할인 여부 판단

크리스마스 디데이 할인
- 방문 날짜가 크리스마스 디데이 이벤트 기간(1~25)인지 판단한다.
- 방문 날짜에 따른 디데이 할인 금액을 계산한다.

평일 할인
- 방문 날짜가 평일(일요일~목요일)인지 판단한다.
- 주문에 포함된 디저트 메뉴 개수를 기준으로 할인 금액을 계산한다.

주말 할인
- 방문 날짜가 주말(금요일,토요일)인지 판단한다.
- 주문에 포함된 메인 메뉴 개수를 기준으로 할인 금액을 계산한다.

특별 할인
- 방문 날짜가 (3, 10, 17, 24, 25, 31)일의 경우 판단한다.
- 총 주문 금액에서 1000원을 할인한다.

이벤트 기간은 크리스마스 디데이 할인을 제외한 다른 이벤트는 1일 부터 31까지 적용한다.

4. 증정 이벤트 여부 판단
- 할인 전 총 주문 금액이 12만 원 이상인지 판단한다.
- 증정 메뉴 제공 여부를 결정한다.
- 증정 금액을 계산한다.

5. 혜택 및 결제 금액 계산
- 적용된 할인 금액의 합계를 계산한다.
- 총혜택 금액(할인 금액 + 증정 금액)을 계산한다.
- 할인 후 예상 결제 금액을 계산한다.

6. 12월 이벤트 배지 부여
- 총 혜택 금액에 따른 이벤트 배지를 결정한다.
- 5천원 이상 : 별, 1만원 이상 : 트리, 2만원 이상 : 산타

7. 출력 데이터 구성
- 주문 메뉴 출력 정보를 생성한다.
- 해택 내역 출력 정보를 생성한다,
- 증정 메뉴가 없는 경우를 처리한다.
- 이벤트 배지가 없는 경우를 처리한다.

## 프로그램 실행 흐름

1. 방문 날짜 입력 받는다.
- 문자열을 통해서 입력 받고 validator를 통해 검증한다.
- 입력 형식이 옳바르지 않을 경우 에러와 함께 입력을 다시 받는다.

2. 주문할 메뉴와 개수 입력
- 주문할 메뉴와 개수를 가지는 문자열을 입력받는다.
- 문자열 형식을 검증한다.
- 입력 형식이 옳바르지 않을 경우 에러와 함께 입력을 다시 받는다.

3. 주문 목록 작성
- 방문 날짜, 주문 메뉴를 Paser를 통해서 파싱하고
- 주문 목록 도메인 Order를 반환한다.

4. 이벤트 참여 여부 결정 및 혜택 적용
- 주문 정보 Order를 통해서 이벤트 참여 여부를 검증한다.
- 주문에 맞는 혜택을 적용하고 혜택 도메인 Event를 반환한다.

5. 상세 내역 출력
- 주문 메뉴 출력 정보를 생성한다.
- 해택 내역 출력 정보를 생성한다,
- 증정 메뉴가 없는 경우를 처리한다.
- 이벤트 배지가 없는 경우를 처리한다.
